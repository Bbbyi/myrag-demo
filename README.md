# myrag-demo
A RAG (Retrieval-Augmented Generation) demo for local private knowledge base Q&A system

## 一、核心技术栈
| 分类         | 具体内容                                                                 |
|--------------|--------------------------------------------------------------------------|
| 核心技术     | Python、自然语言处理（NLP）、大模型微调（LoRA）、向量检索（FAISS）、光学字符识别（OCR） |
| 框架/库      | LangChain、Transformers、PaddleOCR、Gradio、jieba、HuggingFace Embeddings          |
| 工具/能力    | 文本语义分块、混合检索（向量 + 关键词）、大模型推理、自定义 Prompt 工程、知识库构建与管理 |

## 二、项目简介
基于 Python 构建的私有化本地知识库问答系统，支持 PDF/图片/TXT 多格式文件上传，结合 NLP 与大模型技术实现精准问答，所有数据全程本地化处理，保障信息安全与隐私，CPU 环境即可轻量化部署。

## 三、核心功能
### 1. 知识库构建
- **多格式文件解析**：适配图片（PaddleOCR 解析）、PDF（PyPDF2 解析）、TXT（多编码自动兼容），解决不同格式文本提取乱码、空白、识别失败等问题；
- **语义分块与主题提取**：基于 LangChain 实现中文语义边界分块，结合 KeyBERT/正则混合策略完成主题关键词提取，提升检索精准度；
- **向量库管理**：采用 FAISS 构建本地向量库，实现 Embedding 向量的存储与检索，内置维度校验、损坏库自动备份等容错机制，保证系统稳定性。

### 2. 问答核心能力
- **混合检索策略**：设计「向量相似度 + 主题匹配 + 关键词过滤」三层过滤机制，大幅提升检索结果的相关性与精准度；
- **大模型适配**：基于 Qwen2-0.5B-Instruct 模型，通过 LoRA 轻量化微调定制垂直场景问答能力，兼顾效果与部署成本；
- **智能问题处理**：实现「规则前置 + 大模型」的多问题拆分逻辑，支持批量问题解析，避免单一问题过度拆分；
- **回答标准化**：自定义 Prompt 工程约束回答规则，结合结果清洗逻辑，确保输出格式统一、无冗余信息。

### 3. 交互层实现
- 基于 Gradio 搭建可视化操作界面，支持文件上传、知识库分类管理、问答结果展示等核心功能；
- 全链路异常捕获与日志记录，方便问题定位与系统维护。

## 四、核心优势
- **私有化部署**：全程本地运行，数据不联网、不泄露，满足隐私保护需求；
- **轻量化适配**：CPU 环境即可部署运行；
- **高兼容性**：支持多格式文件解析，适配宠物、菜谱等多类垂直场景；
- **高鲁棒性**：内置多重容错机制，降低文件解析、向量检索、模型推理环节的异常风险。
